Nine-lane group-design panel (learning-design [chair], rhythm-meter, virtuoso-ux, gamification, guitar-pedagogy, bass-pedagogy, sound-design, feedback-compatibility, devops-operability). ## The frame (L&D chair + rhythm-meter)
The leading count-in, the inter-block breath, the loop-wrap, and the resume re-count are **ONE primitive** — *"establish the incoming block's pulse (+ key) before its notes arrive."* The engine already does this for single drills; **the Workout (session) path just never fed it.** Most fixes are 1–2 lines of plumbing.

## HOST CHECK (host-expert; prior "no host count-in" memory was stale — corrected)
The host now ships a count-in, an animated loop-wrap, and a Section-Practice bar — **all host-player-bound**, so a CONTAINED plugin can't invoke them. Verdicts:
- **Count-in / loop-wrap → MIRROR** the host's defaults: 4 beats, accented downbeat, **count at the INCOMING block's tempo**, center number; wrap = visible rewind + count + off-target guard (never a hard cut). The host independently arriving at "count-in on loop/section re-entry" validates our inter-block-break = count-in model.
- **Section names on the highway → BORROW (free):** highway_3d's "Now / Up Next + countdown" HUD is default-ON and reads `bundle.sections` (we already feed per block) — confirm names are human-readable.
- **Do NOT route through `window.slopsmith.setLoop/playback`** — host-player-bound; would break contained playback.

## Tier 1 — BUILT + VERIFIED (2026-06-09, plugin.json 0.7.8-dev)
All in `screen.js`/`screen.html`; net **16/16** (smoke-session-sync gains a 5-assert "Workout-love Tier 1" guard block: count-in lead, breaks + wrap, `auto` 1-bar floor, `off`, alt-tuning).

1. **Count-in design change (Christian's call):** the **0-bar "drop-in" is removed from the product UI** — it's a DAW *recording* convenience, not a practice need; a count-in is always-on (≥1 bar). Both dropdowns drop the "None"/"Off" option, default 1; `loadSettingsPrefs`/`applyCountInDefault` floor ≥1. (`readConfig` still *accepts* 0 — the smoke harness re-adds the option for its note[0]-at-t=0 fixtures; players can't reach it.)
2. **Block-1 count-in:** `onLaunchSession` sets `session.countInBars = countInDefaultBars()`; `generateSession` carries it into `sessionMeta` (core-pure, floor ≥1) → `makeBundle`/`applyCountIn` prepends it, counted in **block 1's meter**. Was dropped (Workouts slammed in cold). Verified: `leadIn > 0`, note[0] at the lead.
3. **Inter-block breath — 1-bar floor (FORK A, L&D + rhythm-meter):** `interBlockBreakBars` returns **1** (not 0) on flowing seams — a felt chunk-boundary + the per-block verdict beat every transition; real transitions keep the quantized ≥2-bar break, **+ a ~3 s absolute floor on a technique-class change or ≥5-fret leap** (guitar-ped: at fast tempos 2 bars starves a legato→slap re-grip). `off`=0.
4. **Break clicks (the "silent break" fix):** the click gate now always-clicks `brk` beats even with the metronome off (the break IS a count-in); the break click sits **−2.5 dB** under the in-block metronome (same timbre → the ear locks the incoming pulse, the next downbeat reads as an arrival).
5. **Loop-wrap breath:** a *trailing* wrap break appended in `buildSessionChart` inside `[0,t]` (devops: keeps the count-in shift + `contentLen` phase-carry correct) — the re-entry count for passes 2+. Verified.
6. **Alt-tuning merge:** `onLaunchSession` threads the player's (already-`applyTuningAdaptL1`-resolved) `customOpenMidis` into the session **only when the instrument family matches**; `buildSegmentConfig` length-guards it per-segment, normalizes it to an **array** (the bug: it was a CSV string, which `openMidisForConfig`'s `Array.isArray` gate ignored — this also fixes L3 rung-coded string tunings like djent), and threads the effective tuning to the **shape resolver** (guitar-ped's resolver-not-just-pitch catch). Verified: drop-D low string = 38/D.
7. **UX:** **"▶ Launch Session" → "▶ Start Workout"** (strip the dead host-handoff "launch" ceremony; "Workout" stays the user word); the length picker (Quick/Standard/Woodshed) moved **inline at the commit point** above Start, reframed as "how long you're settling in for."

## Decision log / forks resolved
- **0-bar count-in removed** (Christian): not a practice feature; default 1 everywhere. Panel unanimously backed "count-in is mandatory." (The default was already 1 via settings; this removes the off-switch.)
- **Breath floor 0→1 (Fork A):** L&D *and* rhythm-meter both recommended the 1-bar minimal floor over the seamless segue; gamification's "don't pad / give it meaning" met by keeping flowing=1 (minimal) + the seam verdict. Adopted.
- **applyLengthPreset add-arc — DEFERRED to Tier 3** (L&D's "biggest pacing gap": it scales blocks longer = mindless massing; should add a Review pass / 2nd application). It's content-redesign, not a bug-fix.

## Tier 2 — BUILT + VERIFIED (2026-06-09, plugin.json 0.7.8-dev)
The "establish the incoming pulse" primitive now extends to **resume** and gets a **visible readout** so a silent breath never reads as dead air. All in `screen.js`; net **16/16**; resume verified by `probe-resume-preroll.mjs` (6/6), the visuals by `shot-seam-visuals.mjs` (`shot-seam-countin.png` / `shot-seam-overview.png`, zero page errors).

1. **Resume count-in (pre-roll):** pause→resume snaps the playhead BACK to the re-entry bar downbeat and counts in 1–2 bars before the notes return (notes muted on the runway, the count clicks forced) — the same "don't slam in cold" rule applied to a pause. Implemented as a transport-side `_preRollUntil` window: `resumePlayback` computes the re-entry downbeat ≤ the pause point, `preBars=clamp(round(2/localBar),1,2)`, sets `currentPracticeTime`/`anchorPlayClock` to the runway start and `_preRollUntil` to the re-entry; `tick` clears it on arrival. **Both clock anchors are offset together** (the highest phase risk — `anchorPlayClock` re-anchors all three domains, so no added A/V drift: probe steady-skew before≈after). Notes muted (`n.t < _preRollUntil`) + verify/pitch frames discarded on the runway (re-orienting, not playing).
2. **Seam visuals — the countdown readout (`seamCountInfo` + `drawRulerFrame`):** the head count-in, the inter-block break, the loop-wrap break, and the resume runway are ONE primitive, so they share ONE readout. While counting, the working ruler washes the runway ahead of the playhead amber, the loop strip shows a label (`COUNT-IN` / `NEXT ▸` / `RESUME`), and a **countdown digit (4 3 2 1)** sits over the runway — counted at the incoming pulse (the brk/lead beats are built in the incoming block's meter). Clears on the downbeat; suppresses the "LOOP · drag here" hint while active. Directly answers gamification's "the breath carries no readout = dead air."
3. **Seam visuals — overview rest bands (`drawOverviewFrame`):** the inter-block breaths and the trailing loop-wrap breath aren't in `segmentBounds` (they're the GAPS between bands), so they read as void. Paint each gap a neutral slate with a centered breath glyph (`·` inter-block, `↻` the loop-wrap) so the whole-session map shows the rests as deliberate beats. Finite only (a jam's overview stays the chord loop).
4. **Visible-rewind wrap animation (`_wrapAnim` + `triggerWrapRewind` + `drawOverviewFrame`):** on a loop wrap the overview playhead used to teleport from the seam-out point back to the re-entry point. Now it mirrors the host's animated loop-wrap with a brief **draw-only "tape rewind"** sweep — the playhead whips back (ease-out, ~300 ms) from the seam-out x to the live re-entry x, with a fading red motion streak trailing toward the seam. Fired from BOTH wrap sites (the seamless whole-chart wrap *and* an A–B loop); the clock/audio already wrapped seamlessly, so this is purely the visible tell. **Never touches the transport** (drift-free). Gated: `prefers-reduced-motion` suppresses it, and a **2.5 s min-span gate** keeps a short A–B loop from strobing. The working ruler keeps its fixed-playhead scroll (the overview is the "tape" surface where a playhead travels the whole session). Verified `probe-wrap-rewind.mjs` (6/6: wraps, arms with `fromT` ≈ seam-out, clears at ~280 ms, reduced-motion suppresses, no errors) + `shot-wrap-rewind.mjs`.
5. **Count-in `Off` removed from the transport bar too:** Tier 1 dropped the 0-bar option from both dropdowns but the transport segment group still exposed `data-countin="0">Off`. Removed for consistency (the 0-bar drop-in is gone from all product UI; `readConfig` still *accepts* 0 for the smoke harness only).

### The convergence (resolves the v1 scope fork)
`sessionEnd()` (screen.js:16866) scores the **whole Workout as ONE continuous run** — there is no per-block hit/miss tracking. So any per-chapter or per-seam **score would be fabricated**. All three of L&D, gamification, and UX independently landed on the same answer: **v1 is DESCRIPTIVE / metadata-only** — completion, named competency, the arc traveled — which is exactly what the no-aggregate-% rule already wants. The earned-green "✓ proved" path is **wired but supplied `null` in v1** → it lights up only in **v2**, when a per-block flip-capture (`_activeSession.blocks[] = {name, role, earned|null, hadInput}` captured at each seam) exists. v2 is gamification's follow-on; v1 needs no scoring.

### Piece 2 — per-block SEAM VERDICT (split past/future by tense across surfaces)
- **Past (just-finished block) → a DOM "breath card"** in the stage lower-third (`pointer-events:none`, clear of the bottom-right per design-system §8; reuse `ptDodgeOverlays`): line 1 dim `✓ [competency] — done` (completion, **never a score**); line 2 bright `Next ▸ [next competency] · block i of N` (position breadcrumb). Crossfade in 160ms at break start, out 200ms as the count clears; **reduced-motion = instant**; **no audio** (the break already clicks). Only in `mode==='session'` inter-block breaks — not a single-drill head count-in, loop-wrap, or Jam.
- **Future beat-count → stays on the canvas ruler** (Tier-2 `NEXT ▸` label + countdown digit). The **card owns the names, the ruler owns the digit — never duplicate**. While here: add the incoming block **name** to the ruler `NEXT ▸` label (specced in the Tier-2 charette, never built).
- **Breath sizing (rhythm-meter):** when a verdict renders, `bars = min(4, max(b0, ceil(2.8/barSec)))` — a read bar + a count bar (1 bar only suffices ≤~96 BPM); `b0 = interBlockBreakBars` floor; bump **only** when a verdict shows; the wrap break is the last block's verdict seam. **Single `brkBars` source** per seam (feed both the `buildBeats` push and the `beatOff/barOff` advance).
- **UX-flagged fix:** over a 3–4-bar verdict break the countdown digit must derive from **beats-to-downbeat in the final bar**, not a raw beat count (else "12 11 10…").

### Piece 3 — `applyLengthPreset` ADD-ARC (L&D curriculum + rhythm-meter time)
Replace the proportional `targetSec` stretch with **`composeArc(session, preset)` (structurally ADD blocks) → `doseToFit(arcSession, totalSec)` (per-block dose with a ceiling)**, returning a **new session with reordered+extended `segments[]` — NEVER a chart splice** (composes with the inter-block breaks / `segmentBounds` / per-block backing / wrap for free, and keeps `smoke-session-sync` + `smoke-variation` coverage).
- **Budget split (rhythm-meter):** `cap = {quick:1.25, standard:1.5, woodshed:1.75}` (most the original arc may stretch). `originalBudget = min(total, natSum·cap)`; `addBudget = total − originalBudget`. `addBudget ≤ 0` → **no add-arc** (clean degrade to today's proportional fill). Else original blocks `targetSec = (nat_i/natSum)·originalBudget`; added blocks `(nat_j/rsum)·addBudget`, each also capped; all through `fillBlockToDuration` (whole cells, never cut).
- **Per-category dose ceiling (L&D, the forcing function):** motor ~90s · cognitive ~60s · application/jam ~120s · warmup ~75s · cooldown ~60s — once a block hits its ceiling, extra time can't stretch one run; it flows to more contexts then more blocks.
- **Extra-time ladder:** (1) dose blocks to category length → (2) widen via more contexts (Travel; conservative v1 = bump a block's keyCycle where it already supports one) → (3) interleave a spaced **Review** (re-roll/transpose of the primary `technique`/`scale_arp` block, `role:'review'`, **before the application/jam block if present, else appended**) → (4) a transposed **2nd application** (clone the last application/jam, next circle-of-5ths key) → (5) extend the closing jam.
- **Tier map:** **Quick (~10)** = dose only, no added blocks · **Standard (~20)** = dose + contexts + a closing Review *only if it ends cold* · **Woodshed (~30)** = the full ladder.
- **Derivation (always derive, never invent):** re-roll `variantIdx` → next vary-cycle key/shape → transpose (circle-of-5ths) for inline slots → **never a session-wide BPM bump** (Speed is the depth-ladder's Climb axis). Optional single capstone speed nudge on the final Woodshed application only (opt-in, deferred).
- **Correctness (rhythm-meter):** recompute from the **canonical original arc** — tag added blocks `_added:true` + `role:'review'` and **strip them before recompute**, or reviews compound on every preset change. Non-mutating (returns a new session; never touches `BUILT_IN_SESSIONS`/`_workoutDraft`).

### Decisions / forks resolved
- **v1 descriptive (no per-block scoring)** — unanimous; the per-block earned-signal capture is v2.
- **Review placement** — L&D (interleave for spaced retrieval) vs rhythm-meter (append, fewer seams): **resolved** → interleave before the application/jam block when present (≤1 insertion), append when the session ends cold. Always a reordered `segments[]`, never a chart splice.
- **3D Highway seam card line 2 vs host "Up Next" HUD** — default keep both (the card adds the `block i of N` breadcrumb the HUD lacks); one-line tunable to suppress on 3D if it reads busy.
- **Seam-verdict breath BUMP — DEFERRED in the as-built (main-thread call).** Rhythm-meter's `ceil(2.8/barSec)` read+count bump was premised on the verdict *colliding with the count digit on the shared ruler surface*. UX's design put the verdict on a **separate DOM surface** (the stage breath card), so the collision premise no longer holds, and bumping every inter-block seam would supersede Tier-1's deliberate 1-bar flow floor (and its `smoke-session-sync` guard) and lengthen sessions. Gamification also rules the card an *optional glance* ("missing it costs nothing"). So v1 ships the card in the existing Tier-1/2 breath window; the **countdown-digit fix is kept** (an unambiguous bug — a multi-bar break read "12 11 10…"; now counts within the bar). If dogfood shows the card is too brief at fast tempo, the bump (gated on a verdict flag through `interBlockBreakBars`) lands then.
- **add-arc step 2 (within-block context widening)** — underspecified by the panel; v1 ships the conservative form (cap stretch + funnel excess to added blocks; bump keyCycle only where supported), full Travel-widening deferred.
